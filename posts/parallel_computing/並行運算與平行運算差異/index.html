並行運算與平行運算差異

July 4, 2022

並行運算 ( Concurrent computing ) 與平行運算 ( Parallel computing ) 名稱相似，但兩者意含不同。並行運算著重在系統允許多項任務在一定時間內同時進行；平行運算強調藉由分治 ( Divide and conquer) 提昇執行任務的效能。

並行運算

並行運算允許多項任務在一定時間內同時存在並交錯進行，一項任務可以不等待所有其他任務完成逕自執行。與並行運算相反的概念是順序運算 ( Sequential computing ) ，順序運算代表在當前任務完成前，其他任務不能被執行。

圖二、順序運算

程序 ( Process ) 並行運算體現在作業系統採用時間分割 ( Time slicing ) ，使的多項程序同時存在並交錯執行；執行緒 ( Thread ) 的並行體現在單一程序實踐多執行緒。

值得注意的是，任何一種形式的並行運算，都必須避免競爭危害 ( Race condition )。競爭危害可以被定義為：「由於事件在相對時間下的不預期重大依賴導致的異常行為」(原文出自 Avoid Race Conditions )。例如，多項程序嘗試修改同一份文件，但由於並行運算導致文件的最終結果不符合原先預期。
圖三、由於競爭危害導致文件結果可能不如預期

為了避免競爭危害，採用並行運算應該同時考慮妥善控制機制。

平行運算

平行運算採用分治法 ( Divide and conquer ) 通常會將一項任務拆分成多項相似的子任務，每項子任務被分配到不同的運算單元上，彼此獨立執行互不影響，再將子任務的執行結果合併得到最終任務結果。由於平行運算利用多組運算單元，因此比只指利用單一運算單元，平行運算能夠提昇執行任務的效率。

圖五、平行運算

平行運算倚賴硬體支援，例如存在多組運算單元的單一設備上進行平行運算，或是在能夠彼此溝通的多台具運算能力的設備上進行。

儘管有多組運算元，運算元數量與執行效率並未呈現線性關係，阿姆達爾定律 ( Amdahl’s law ) 描述兩者之間關係，造成此現象的其中一原因在於運算元數量越多所造成的溝通成本越高，可看出平行運算的瓶頸在於如何溝通協調各組運算元。

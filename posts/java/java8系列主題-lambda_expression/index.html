Java 8 - Lambda Expression

April 21, 2022

Lambda Expression (Lambda 表達式) 的出現使的開發人員能夠用更簡潔的語法達到 Anonymous Classes (匿名類別) 所能達到的功能，前提是原有寫法的 Anonymous Class 裡只有單一 method。

讓我們先快速看過何謂 Anonymous Classes，假設現在有一 interface PersonCheckInterface 宣告如下：

interface PersonCheckInterface {
    public boolean check(Person person);
}

有一段程式碼試試圖對 instantiate (實例化) interface PersonCheckInterface 並實作 method check

PersonCheckInterface personCheck = new PersonCheckInterface() {
    public boolean check(Person person) {
        return (person.getAge() >= 20) ? true : false;
    } 
}

上段程式碼中產生了 Anonymous Class ，這個 Anonymous Class implements PersonCheckInterface 且為 interface 下 的 method 提供實作細節。在 Oracle Java Tutorials 對 Anonymous Class 包含哪些要素講的十分清楚，包含：

  • New operator
  • 被 implements 的 interface 或是 extends 的 class 名稱
  • 括弧，括弧內部包含需要傳入 Constructor 的參數，如果不需要參數可以僅用空括弧 () 表示
  • Declaration body，和宣告一般 Class 裡的內容同樣

了解 Anonymous Class 之後，回歸正談，讓我們透過 Lambda Expression 精簡上述程式碼。由於 PersonCheckInterface 只有 宣告單一 method ，因此上述程式碼使用 Lambda Expression 改寫後如下：

PersonCheckInterface personCheck = person -> {
    return (person.getAge() >= 20) ? true : false;
};

和之前的程式碼相比，Lambda Expression 不需要 instantiate ，因此如果你有注意會發現使用 Lambda Expression 後 Compile 產生的 .class 檔會比原本使用 Anonymous Class 寫法產生的 .class 檔少。從這裡可以明白使用 Lambda Expression 的好處。

Lambda Expression 的語法可以表示為

  • 括弧包含 Override Method 裡的參數，如果只有單一參數，可以不用括弧。以上述為例， method check 只會接收單一參數 Person person， 因此不括弧包含的寫法 (person) ，使用 person 僅可。
  • 箭頭符號 ->
  • 主體，包含單一 expression 或是 statement block。以上述為例，使用 statement block 明確表示 method check 的實作應該 如何 return。如果實作的 method return type 為 void 並且只有一行 expression ，可以大括弧。舉例，假設 method check return type 現在是 void ，並且我想要印出 person.getAge() 資訊，上述的程式碼將會是
PersonCheckInterface personCheck = person -> System.out.println(person.getAge());
